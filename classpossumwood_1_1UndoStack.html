Detailed Description

A simple implementation of undo stack based on Command design pattern. Each command uses a vector of std::functors to store the actual code to be executed on each undo and redo.

void possumwood::UndoStack::execute ( const Action &  action)

Executes an action and puts it to the undo stack to make it undoable. Any potential exceptions thrown from an action are handled (assuming they are based on std::exception) - an error during a command will assume that command has not been finished (and any remaining state is rolled back), and all previously finished commands of an action needs rolling back. Exception is then forwared to the caller.

void possumwood::UndoStack::redo ( )

Redoes last undone action. The redo queue gets cleared when a new action is inserted onto the stack.

void possumwood::UndoStack::undo ( )

Undoes last succesfully executed action. Cannot throw - action worked last time, so it should work this time as well.
